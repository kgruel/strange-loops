# tasks — task orchestration (archived)

> **Note:** This app was an early experiment in building task orchestration on top of the strange-loops system. It is not actively maintained and should not be treated as a current example of how to build on the libs. The patterns here predate several architectural changes to the core system.

Tasks are loops. Workers run in worktrees. The store is the coordination channel.

See the `libs/` and `apps/loops/` for current usage patterns.
